
打包生成的apk里面东西很少的，不包含android sdk。


你不是在Android设备上开发Android应用。
你的开发环境中并没有你需要的系统接口，你需要的依赖。
SDK即Soft Development Kit，谷歌为了让你在非Android环境上能够开发Android应用而提供的一系列开发套件。
它提供了和真机环境下一致的类，方法，属性，接口等，让你能够在非Android环境下顺利编译通过。当你编译的apk在真机上运行时，调用的才是真的被实现了的接口。


1. Android APP 打包包含 Android SDK 吗？不包含，Android sdk(framework)是手机里自带。及时强制打进去了，也是用的手机里的。
2. Android 系统打包包含AndroidSDK 吗？当然包含，在手机系统里
3. SDK 是个什么东西，为什么需要SDK，而不是系统提供出接口直接调用？到底起个什么作用？直接在 APP 里写代码调用系统一系列接口不可以吗？为什么要用 SDK？其实现在就是直接调用系统的接口。
开发时引用的sdk，实际也只是空壳，为了帮助编译和代码提示。另外有些实现是在系统进程里的，进程数据隔离，直接没法调用，需要跨进程通信，sdk也帮助封装了跨进程通信IPC的代码，方便调用。
4. 为什么 SDK 里很多代码报红依然能通过编译，而 APP 里引用了 SDK 里同样报红的代码就不行？sdk的是编译好的class，是供app编译时引用的，实际不会编译。标红是IDE没那么智能，找不到就会报红。
APP里放同样的代码，不一定能直接引用到其他相关类呀，例如一些隐藏类，或者这个类本身就是系统进程里的。自然不行了


Android系统是一个以Linux为内核的操作系统，在Android系统中搭建了一层虚拟机即DVM或ART，为了便于开发者开发APP所以在这层运行时上面搭建了一层框架即frameworks，
从这层frameworks以及其他一部分代码中抽出来了一部分接口提供给开发者即SDK从而使我们可以编译APK或AAR。
你所说的“SDK里很多代码报红依然能通过编译”但是实际上你编译自己的APK时依赖的并不是你看到的这部分代码而是已经编译完成的SDK，你并没有编译过SDK本身。
甚至你所看到的“SDK里很多代码”甚至可能是IDE给你反编译出来的。
